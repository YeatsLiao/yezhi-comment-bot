# 叶芝说 SKILL 版本更新日志

## v2.9（当前版本）— 2026-04-18

**备选标题增加且要求紧扣核心信息**

旧版问题：
- 备选标题只提供3-5个，数量不够
- 部分备选标题空泛（如"AI上机了"），没有抓住文章具体内容
- 备选之间只是同义替换，角度单一

新版改为：
- **备选标题增加到5-8个**
- **每个备选必须紧扣文章核心信息**（具体事件/数据/冲突）
- **每个备选要体现不同切入角度**（数据切入、冲突切入、用户利益切入等），不能只是同义替换
- 示例（Codex操控电脑一文）：
  - ❌"AI上机了"（空泛）
  - ✅"给AI两个权限它就能控你的电脑"（具体信息+冲突）
  - ✅"OpenAI想当你的电脑管家"（品牌+角色转变）
  - ✅"你的屏幕AI全能看到"（用户利益+隐私担忧）

## v2.8 — 2026-04-18

**标题规则收紧**

旧版标题允许15字、允许带逗号、风格偏书面。

新版改为：
- **一句话、不带逗号、不超过12个字、口语化、像跟朋友说话**
- 示例：❌"AI不再只跟你聊天了，它开始替你干活" → ✅"AI开始替你上班了"
- 示例：❌"被英伟达卡了这么多年脖子，终于断了" → ✅"英伟达的脖子被掐断了"

## v2.7 — 2026-04-13

**新增交付时提供备选标题**

旧版交付只展示一个选定标题，用户如果想换标题需要重新想。

新版改为：
- **交付时提供备选标题（必须执行）**：文章完成后额外提供3-5个不同风格的备选标题
- 备选来自头脑风暴落选的优质候选，或基于正文新提炼
- 用户可随时替换，无需重新生成整篇文章

---

## v2.6 — 2026-04-12

**新增标题头脑风暴步骤**

旧版写标题是"想一个直接用"，质量依赖AI的即时发挥，容易产出平庸标题。

新版改为：
- **标题头脑风暴（必须执行）**：选定话题后，按6个公式各写2-3个候选标题（共12-18个），从中选最佳
- 筛选标准：有具体信息、有悬念、跟读者有关、口语化、不超过15个字
- 自检："如果我在朋友圈刷到这个标题，我会点进去吗？"

## v2.5 — 2026-04-11

**封面图风格回调：保留digital_art画风，配色轮换机制延续**

v2.4的扁平矢量插画风格经过实际生成测试，效果不够理想。用户决定回调到v2.3的digital_art风格。

变更内容：
- **风格**: 回调为digital_art（现代、干净、专业的编辑插图风格）
- **配色**: 保留v2.4引入的两套配色轮换机制（暖色系/冷色系），根据文章情绪选择，不连续使用同一套
- **提示词模板**: 更新为digital_art风格+配色轮换的组合模板

---

## v2.4 — 2026-04-10

**封面图风格全面升级：从暗色数字艺术到扁平矢量插画**

旧版封面图统一使用深海蓝+暗蓝+钴蓝+亮红的暗色调digital_art风格，文章多了之后视觉同质化严重。

新版改为：
- **风格**: 扁平矢量插画（flat vector illustration），类似The Economist、Bloomberg的编辑插画风格
- **两套配色轮换**：
  - 暖色系（白底）：珊瑚红+琥珀黄+青绿+炭灰，适合生活/消费类话题
  - 冷色系（深色底）：电光蓝+品红+明黄+深海军蓝，适合科技/商业/争议性话题
- **提示词模板**: 分别提供暖色系和冷色系两套完整模板

## v2.3 — 2026-04-07

**热点搜索范围扩展到全球**

旧版搜索策略只覆盖国内源（微博、36氪、IT之家），容易错过国际科技大事件。

新版改为：
- **搜索策略升级**：国内热点（微博热搜）+ 国际热点（tech news today）双线并行
- **信息源扩展**：新增 Twitter/X、Reddit、TechCrunch、The Verge、Ars Technica、Hacker News、Bloomberg Tech、Reuters Tech 等国际源
- **语言不限**：国际科技新闻同样可以评论，尤其是对国内读者有启发或影响的事件

## v2.2 — 2026-04-07

**标题规范全面升级：从"短而有力"到"让读者想点进来"**

旧版标题规范只有3条禁止项（不要对称句式、不要逗号分两句、12字以内），过于抽象，AI产出的标题质量参差不齐。

新版改为：
- **好标题的四个特征**：有具体信息、有悬念或冲突、跟读者有关、口语化
- **6个标题公式**（每次随机选一个，不要连续用同一个）：
  1. 数字+悬念 → "一天1.4万亿"
  2. 品牌+反常行为 → "小米涨价了"
  3. 你+利益相关 → "你的下一部手机，可能已经是AI手机了"
  4. 场景+意外转折 → "AI上车，然后趴窝了"
  5. 提问式 → "AI开始有自己的朋友圈了"
  6. 短陈述+留白 → "连续五周，反超"
- **标题禁忌**：AI味句式、论文式标题、标题党、纯抽象概念、超过15字
- **标题自检**："如果我在朋友圈刷到这个标题，我会点进去吗？"
- 自检清单新增标题检查项

## v2.0 — 2026-04-04

**去AI化写作规范（重大升级）**

这是最重要的一次升级，针对平台"低价值AIGC内容"审核规则全面改造：

| 维度 | v1.0 | v2.0 |
|------|------|------|
| 结构 | 固定"引入→分论点→写在最后" | 随机组合开篇/展开/结尾方式 |
| 语气 | 大量加粗、破折号、"这意味着" | 加粗≤3处，禁用AI高频词 |
| 信息密度 | 观点多干货少 | 每篇≥3个信息增量点 |
| 结尾 | 金句收尾+"写在最后" | ≤3句话，允许无结尾 |
| 标题 | "XX，YY"对称句式 | ≤12字，随机句式 |
| 排比 | "不是A而是B"连续排比 | 禁止连续排比填充 |
| 自检 | 无 | 8项去AI化自检清单 |

新增内容：
- 6条去AI化写作规范（结构/语气/信息密度/观点/结尾/标题）
- 8项自检清单
- 好文章 vs AI文章对比示例
- 写作策略随机组合（开篇4种 × 展开4种 × 结尾4种 = 64种组合）
