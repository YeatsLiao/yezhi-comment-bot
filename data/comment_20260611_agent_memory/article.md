# Agent失忆，不是因为记性差

![封面](cover.jpg)

你有没有遇到过这种情况：让AI Agent查资料，它翻了三页网页，问到第四页的时候，突然忘了第一页看到的关键数据。不是偶尔，是常态。

这不是模型不够聪明，是训练方式出了盲区。

中科大和上海AI Lab刚发了一篇arXiv论文，提出了一个叫ACC（Agent Context Compilation）的方法。说白了，就是把Agent执行任务时产生的多轮轨迹——搜索了什么、调了什么工具、拿到了什么结果——全部打包成一长段上下文，让模型一次性读完，直接给出答案。不用一轮一轮地调工具。

## 一个被忽略的盲区

现在的Agent训练有个默认做法：模型调用工具拿到结果后，这个结果在训练时被mask掉，只让模型学习"下一步该调什么工具"。

论文里管这叫"监督盲区"（supervision blind spot）。证据散落在多轮对话里，但模型从来没被训练过去整合这些分散的证据。它只学会了局部决策——这轮该搜网页还是该写代码——却没学会全局推理。

打个比方。你让一个人去图书馆查资料，他每次查完一本书就向你汇报"我查了这本书"，但从不告诉你书里写了什么。查完十本，你让他总结，他只能凭印象瞎编。这就是现在大多数Agent的真实状态。

ACC的做法很直接：把十本书的内容全部摊在桌上，让人一次性看完再回答。训练目标从"学会调工具"变成了"学会在长篇材料里找答案"。

![](https://images.unsplash.com/photo-1709120395858-92f1c7c577f5?w=1200)

## 效果有多夸张

论文在Qwen3-30B-A3B上做了实验。这个模型不算大，30B参数，带3B激活。

用ACC训练后，MRCR（多轮共指消解，测的是跨轮次关联信息的能力）从50.2飙到68.3，涨了18.1分。GraphWalks（图遍历，测的是在超长上下文里追踪关系的能力）从69.9涨到77.5。

什么概念？30B的小模型，长上下文能力追平了235B的Qwen3-235B-A22B。差了将近8倍的参数，结果打平。

更关键的是，通用能力没掉。GPQA、MMLU-Pro、AIME、IFEval这些标准benchmark，分数和原来基本持平。不是拿通用能力换专项能力，是真学会了新本事。

论文还做了机制分析，发现ACC训练后的模型出现了两个现象：一是注意力结构会随任务自适应重组，不同任务激活不同的注意力模式；二是专家路由出现了专业化倾向，某些专家专门处理远距离依赖，另一些专门处理局部信息。这暗示模型内部真的形成了分工。

![](https://images.unsplash.com/photo-1644325349124-d1756b79dd42?w=1200)

## 为什么这事值得认真看

ACC的核心价值不是提出了一个多厉害的算法——它本身很简单，就是把轨迹拼起来做SFT。它的价值在于指出了一个被整个行业忽略的方向。

Agent产生的轨迹数据是海量的。Claude Code、Cursor、各种coding agent，每天都在产生成千上万条多轮执行记录。这些数据以前基本被浪费掉了，最多拿来做点偏好对齐。ACC告诉你，这些轨迹本身就是最好的长上下文训练数据，不需要额外标注，不需要请人写长文档。

而且ACC跟现有方法不冲突。你可以先拿ACC做SFT，再上longRLVR或LongPO做后训练，效果只会更好。论文作者已经开源了数据集和checkpoint，HuggingFace上能直接下载。

另一个值得品味的点是：这篇论文来自中科大和上海AI Lab，不是OpenAI也不是Anthropic。国内机构在Agent基础设施层面的原创工作，正在变多。

## 一个务实的判断

Agent的"失忆"问题，短期内不会被某个银弹解决。ACC提供的是一条低成本、可规模化的数据路径，让模型学会"回头看"。

但真正落地到产品里，还有距离。轨迹编译的质量、噪声过滤、答案验证，这些工程细节论文没展开。而且ACC目前只在搜索、代码、SQL三类Agent上验证过，更复杂的规划型Agent能不能受益，还得看后续工作。

不过方向是对的。与其继续堆参数、扩窗口，不如先把手里已有的数据用好。

![](https://images.unsplash.com/photo-1619410283995-43d9134e7656?w=1200)

Agent不是记性差，是从来没被教过怎么记。
