# OpenAI推理成本降50%：不换芯片，靠什么

![封面](cover.jpg)

6月30日，The Information报道了一则消息：OpenAI工程师通过一系列系统底层优化，把模型推理成本降低了50%以上。不是靠买更多GPU，也不是靠换新芯片，而是靠**把现有服务器的利用率榨到极致**。

这个消息的冲击力在于，它戳破了一个行业惯性思维——推理成本高，唯一的解法就是堆硬件。OpenAI用行动证明，软件层面的优化空间，远比大多数人想象的大。

## 推理成本到底花在哪

要理解OpenAI做了什么，先得搞清楚推理成本的结构。

大模型推理（Inference）和训练（Training）是完全不同的两件事。训练是一次性投入，花几千万美元训一个模型，模型就固定了。推理是持续性支出——每服务一个用户、每生成一个回答，都要消耗算力。

OpenAI每天服务数亿次对话，推理成本就是运营成本。这个成本的构成，大致可以拆成三块：

```
推理成本构成
├─ GPU计算时间（≈60%）
│   └─ 模型前向传播：输入Token处理 + 输出Token生成
├─ GPU显存占用（≈25%）
│   └─ 模型权重 + KV Cache（历史对话缓存）
└─ 通信与调度开销（≈15%）
    └─ 多GPU并行时的数据传输 + 请求排队等待
```

GPU计算时间是大头。模型每生成一个Token，都要把所有参数过一遍神经网络。这个计算量只和模型大小有关，和用户数量无关——一个用户问和一百个用户问，单次的计算量是一样的。

但这里有个关键细节：**GPU不是每时每刻都在算的。**

一个GPU的算力是固定的，比如每秒能处理1000个Token。但如果用户的请求是断断续续的——上午10点涌入一波，凌晨3点几乎没有——GPU在低谷期就是空转的。空转的GPU也在耗电、也在折旧，但没有产生任何价值。

OpenAI的优化，核心就是解决这个**利用率问题**。

## 优化方向一：从批次处理到流水作业

传统的推理服务，处理请求的方式像餐厅的点餐制——来一个订单做一份，做完再做下一份。

这种做法的问题很明显：GPU算完一个请求后，下一个请求可能还没来，GPU只能干等。即使有多个请求同时到达，它们的长度也可能差别很大——一个用户问你好，另一个用户要求写一篇3000字的文章。如果硬塞进同一个批次，短请求算完后要陪长请求一起等，造成大量无效占用。

行业内的解法叫**连续批处理**（Continuous Batching），也叫迭代级调度。核心思路是把调度粒度从整个请求细化到单个Token的生成步骤：

```
传统批处理（请求级）              连续批处理（迭代级）
├─ 请求A: 生成10个Token           ├─ Step 1: A生成1个, B生成1个, C生成1个
├─ 请求B: 生成100个Token          ├─ Step 2: A生成1个, B生成1个, C生成1个
├─ 请求C: 生成50个Token           ├─ Step 3: A完成退出, B生成1个, C生成1个
└─ 全部完成后才开新批次            ├─ Step 4: 新请求D加入, B生成1个, C生成1个
                                   └─ GPU每一步都有活干，不等人
```

每一步生成一个Token后，已完成的请求立即退出，新的请求立即加入。GPU像一条永不停歇的流水线，每个时间步都有计算任务，没有空等。

vLLM——目前最流行的开源推理引擎——就是靠这套机制，把GPU利用率从传统方案的45%提到了85%以上，吞吐量提升2-4倍。OpenAI作为vLLM的核心贡献方之一，在自己的生产环境中把这套机制做到极致，是完全合理的推断。

## 优化方向二：KV Cache的内存革命

连续批处理解决了GPU算力浪费的问题，但还有一个更大的资源黑洞——**显存**。

大模型生成文字时，需要记住前面已经生成的所有内容。这个记忆不是存在硬盘里，而是存在GPU的高速显存（VRAM）中，专业术语叫**KV Cache**（Key-Value Cache）。

KV Cache的大小和对话长度成正比。一个1000Token的对话，KV Cache可能占用几GB显存。更麻烦的是，传统实现方式是按最大长度预分配的——即使对话只有10个Token，也要按2048个Token预留空间。结果就是大量显存被浪费在空位上。

```
传统KV Cache分配                    PagedAttention分页分配
├─ 序列A: 预分配2048位置             ├─ 序列A: 实际用3页（48Token）
│   └─ 实际使用50Token                │   └─ 第1页满，第2页用一半，第3页空
│   └─ 1998个位置空转                 │   └─ 第3页回收给序列B使用
├─ 序列B: 预分配2048位置             ├─ 序列B: 实际用5页（80Token）
│   └─ 实际使用2000Token              │   └─ 动态分配，用多少给多少
└─ 碎片化严重，无法复用               └─ 无碎片，用完即回收
```

PagedAttention的解法是借鉴操作系统虚拟内存管理的分页机制：把KV Cache切成固定大小的小块（比如每块16个Token），按需分配，用完回收。显存利用率从45%提到85%以上，同时能支持3-4倍更多的并发请求。

OpenAI如果在自己的推理集群中全面推行类似PagedAttention的显存管理优化，意味着同样的GPU能同时服务更多用户，单位成本自然下降。

## 优化方向三：请求调度的智能分层

除了计算效率和显存效率，还有一个被低估的优化维度——**请求调度**。

不同用户的请求，优先级和特性差异很大：

- **免费用户**的聊天请求：可以容忍几百毫秒的延迟，对实时性要求不高
- **付费API用户**的调用：通常有严格的延迟SLA（服务等级协议），超时就要赔钱
- **长文本摘要**任务：一次处理几万Token，算力密集但不在乎延迟
- **代码补全**任务：要求毫秒级响应，否则开发者体验极差

如果把所有请求塞进同一个队列，按先来先服务的顺序处理，结果就是高优先级的代码补全请求排在长文本摘要后面干等，用户体验崩盘。

更合理的做法是把请求按特性和优先级分层，分别送入不同的处理管道：

```
请求智能调度分层
├─ 实时层（延迟敏感）
│   ├─ 代码补全、短对话
│   └─ 分配高算力GPU，保证低延迟
├─ 吞吐层（成本敏感）
│   ├─ 长文本摘要、批量处理
│   └─ 允许更高延迟，换取更大批次、更高吞吐
└─ 弹性层（可降级）
    ├─ 免费用户非关键请求
    └─ 高峰期自动降级，优先保证付费用户
```

这种分层调度的效果，不是让单块GPU跑得更快，而是让**整个GPU集群的产出价值最大化**——在同样的硬件成本下，服务更多高价值请求、减少低价值请求的资源挤占。

## 50%成本降低意味着什么

The Information的报道提到，OpenAI在未登录ChatGPT流量上验证了这套优化方案。这部分流量的特点是：请求量大、商业价值相对较低、对延迟容忍度较高——正是软件优化最容易见效的场景。

但更重要的是**信号意义**：

**第一，推理成本的下降空间比预期大。** 行业普遍认为，推理成本的年降速度在20%-30%左右，主要靠模型压缩和硬件迭代。OpenAI用纯软件优化砍了50%，说明现有基础设施的利用率天花板还远没到。

**第二，软件优化的边际成本远低于硬件。** 买一块H100要3万美元，而优化一套调度算法只需要工程师时间。对于已经部署了数十万台GPU的OpenAI来说，软件优化的ROI（投资回报率）是惊人的。

**第三，推理成本的下降趋势会加速。** 当头部公司证明软件优化能带来50%的成本降幅，整个行业都会跟进。vLLM、TGI、SGLang等开源推理引擎已经在快速迭代，未来2-3年推理成本可能再降一个数量级。

## 对行业的影响

OpenAI这次优化的直接影响，可能是**API降价**或**免费额度提升**。The Information的报道明确提到，节省的成本可用于降低API定价或提高用户使用限额。

但更深层的意义在于，它改变了AI应用的经济模型。

推理成本是大模型应用最大的运营支出。如果一个AI客服系统每月推理成本10万元，成本降50%就意味着每月省5万元——这笔钱可以用来做功能迭代、市场推广，或者直接提升利润率。

对于AI应用创业者来说，这意味着：

- **原来亏钱的应用，现在可能盈亏平衡了**
- **原来盈亏平衡的，现在可能有利润了**
- **原来有利润的，现在可以加大投入了**

推理成本的持续下降，是AI应用大规模普及的前提条件。OpenAI的50%降幅，只是这个趋势中的一个节点，不是终点。

```
推理成本下降趋势
2024 ────────────────────────────────────── 基准线
2025 ──────────────────────────────── 模型压缩+量化
2026 ─────────────────────── 软件优化(OpenAI 50%降幅)
2027 ─────────────── 自研芯片(Jalapeño)+软件协同
2028 ────── 推理成本降至现在的1/10
```

当推理成本降到现在的十分之一，现在那些因为太烧钱而无法商业化的AI应用——实时视频生成、全量文档分析、个性化教育助手——都会变得可行。

OpenAI的这次优化，不只是给自己省钱，更是给整个行业拆了一堵墙。
