# AI首次通过编程地狱测试

5月，一个被AI圈称为"编程地狱"的基准测试，第一次出现了满分。

这个测试叫ProgramBench，由Meta Fair、斯坦福和哈佛联合开发。规则很苛刻：只给AI一个编译好的二进制文件和简单使用说明，让它从零写出功能完全一致的代码。测试涵盖200个任务，包括数据库、编译器、命令行工具等各种复杂软件类型。

在此之前，包括Claude Opus 4.7、GPT-5.4、Gemini 3.1 Pro在内的所有一线模型，通过率都是0%。

GPT-5.5拿了满分。

**这意味着什么？**

简单说，以前的AI编程助手是"代码补全工具"——你写个开头，它帮你补后面。现在GPT-5.5展现的是"逆向工程能力"：给它一个可执行程序，它能反推出源代码，而且代码能跑、功能一致。

在cmatrix（终端矩阵雨动画程序）这个任务上，GPT-5.5在高推理模式下分别用C和Python写了两个版本，全部完美通过行为测试。C版本花了3.17美元，Python版本4.84美元。相比之下，表现第二好的Claude Opus 4.7花了10.74美元，调用178次API，还有19个测试没通过。

**成本效率是3倍，完成度是满分对不及格。**

当然，这里有个关键变量：推理强度。在默认模式下，GPT-5.5的表现只比Claude Sonnet 4.6略好；但切换到最高档推理模式后，它就把所有对手甩在了身后。这说明深度思考和规划能力，是这次突破的核心。

**更值得关注的是另一个数据。**

OpenAI内部有个叫Expert-SWE的测试，专门测"中等难度、人类需要约20小时完成"的复杂工程任务。GPT-5.5拿了73.1%的分数。这意味着，面对需要横跨数百个文件、具备极强逻辑连贯性的企业级项目，AI已经能独立完成近四分之三。

Terminal-Bench 2.0的成绩更夸张：82.7%，比GPT-5.4的75.1%提升了7.6个百分点，比Claude Opus 4.7的69.4%高出13.3个百分点。这个测试测的不是单轮答题，而是给模型一个终端环境和一个模糊目标，让它自己规划路径、调工具、写脚本、处理报错、反复迭代，直到任务完成。

**说白了，测的是"把事情做完"的能力。**

但别急着喊"程序员失业"。SonarSource的独立评测显示，GPT-5.5生成的代码虽然安全指标是业内最好的（漏洞密度每百万行75个），但并发和线程相关的bug密度高达每百万行170个。高并发压力测试下，它生成的代码会出现Redis连接池被打满这类问题。

这说明什么？AI在"写出能跑的代码"上已经很强了，但在"写出能扛住极端场景的工业级代码"上还有距离。真实的软件工程不只是把功能堆出来，还要考虑性能、容错、边界情况——这些恰恰是资深工程师的价值所在。

**另一个有趣的发现是代码结构。**

研究人员发现，AI生成的代码普遍呈现"单体化"特征：大量逻辑塞进单个文件，目录结构浅，模块拆分少。这与人类工程师"高内聚低耦合"的实践习惯截然相反。这暴露了当前AI在全局系统规划上的瓶颈——它擅长局部代码生成，但长期软件系统构建能力仍是短板。

不过，这个短板正在被快速补上。GPT-5.5的上下文窗口已经支持到200万Token，在512K到1M的极限区间，信息召回率达到74%，而GPT-5.4只有36.6%。长上下文能力的质变，让AI处理大型代码库成为可能。

**对行业的影响已经开始显现。**

OpenAI自己的财务团队用GPT-5.5审查了71,637页的K-1税表，比去年提前两周完成。市场部门的周报自动生成系统，每周节省5到10小时人力。公关团队搭建了一套评分和风险框架，低风险请求自动走Slack AI智能体处理。

这些案例的共同点是：不是"帮我写代码"，而是"帮我把这个工作流推进到底"。

GPT-5.5的定价也很有意思：Pro版本API价格30美元/百万Token输入，180美元/百万Token输出，比GPT-5.4翻倍。但Token效率提升了30%以上，实际成本未必更高。这种"智能溢价"策略可能会加速AI应用的分层——简单任务用便宜模型，复杂任务用贵但强的模型。

从4月16日Anthropic发布Claude Opus 4.7抢走编程第一，到4月24日GPT-5.5上线夺回王座，中间只隔了8天。这种迭代速度本身就在说明：AI编程能力的竞争，已经从"谁能写代码"进入了"谁能写系统"的阶段。

ProgramBench的满分，是一个里程碑，但绝不是终点。它标志着AI从"代码片段生成"向"完整软件系统构建"的能力跃迁已经开始。接下来要解决的，是模块化架构、极端场景优化、长期维护性这些更难的命题。

对于程序员来说，好消息是：AI还没法完全替代你。坏消息是：它正在快速逼近那个临界点。
