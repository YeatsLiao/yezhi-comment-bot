# 美团LongCat-2.0：1.6万亿参数全程国产卡训推，技术架构全拆解

![封面](cover.jpg)

2026年7月6日，美团扔出了一颗让国内AI圈震动的炸弹。

LongCat-2.0正式开源——总参数1.6万亿，平均激活约480亿，原生支持100万token超长上下文。更关键的是，这是业界第一个在五万张国产算力卡上完成全流程训练与推理的万亿参数模型。华为昇腾、摩尔线程、沐曦、寒武纪、天数智芯，同一天宣布完成推理适配。

从外卖到AI基础设施，美团的这一步跨度不小。但LongCat-2.0真正的看点不是参数规模，而是它在国产算力受限环境下做的一系列工程妥协与创新。

## 问题定义：国产算力的三重天花板

在理解LongCat-2.0的架构之前，得先看清楚它面对的问题。

国产AI芯片和英伟达H100/B200相比，存在三个客观差距：单卡算力低（约为H100的60%-80%）、显存容量小（主流国产卡64GB vs H100的80GB）、生态工具链不成熟（CUDA迁移成本高）。当模型参数膨胀到万亿级别时，这三个差距会被指数级放大。

具体来说，训练一个1.6万亿参数的模型，在英伟达集群上已经是工程极限。换到国产卡上，问题变成：如何在单卡算力更低、显存更小、通信带宽更窄的条件下，完成同等规模的训练？

美团的解法不是硬扛，而是**用架构设计补偿硬件差距**。

```
国产算力训练万亿模型的三重挑战
├─ 算力密度低：单卡算力 ≈ H100的60-80%
├─ 显存容量小：主流国产卡64GB vs H100 80GB
├─ 通信带宽窄：卡间互联带宽低于NVLink
└─ 结果：同样的模型，国产卡需要更多数量 + 更精细的并行策略
```

## 整体架构：三层补偿设计

LongCat-2.0的核心可以用三个缩写概括：**MoE + LSA + N-gram**。

```
┌─────────────────────────────────────────┐
│  第一层：MoE 稀疏路由（参数效率补偿）      │
│  ├── 总参数1.6T，单次激活约48B            │
│  ├── MoE稀疏度接近97%                     │
│  └── 用激活效率换显存占用                  │
└─────────────────┬───────────────────────┘
                  │ 经路由选择的激活参数
                  ▼
┌─────────────────────────────────────────┐
│  第二层：LSA 稀疏注意力（长上下文补偿）    │
│  ├── 流感知索引（SI）                     │
│  ├── 跨层索引（CI）                       │
│  └── 计算量从O(n²)降至接近O(n)           │
└─────────────────┬───────────────────────┘
                  │ 压缩后的注意力表示
                  ▼
┌─────────────────────────────────────────┐
│  第三层：N-gram Embedding（代码能力补偿）  │
│  ├── 135B参数独立模块                     │
│  ├── 强化代码片段级语义理解                │
│  └── 与MoE专家系统正交互补                │
└─────────────────────────────────────────┘
```

三层设计的共同目标是：**在国产卡显存和算力都受限的情况下，用算法效率换硬件性能**。MoE降低单次推理的显存占用，LSA降低长上下文的计算开销，N-gram Embedding在稀疏度接近极限时额外注入代码能力。

## 第一层：MoE稀疏度的极限博弈

LongCat-2.0的MoE架构和GLM-5.2、DeepSeek-V4等国产模型思路一致：总参数量堆上去，但每次只激活一小部分。

具体到数字：总参数1.6万亿，单次激活约480亿（动态范围330亿到560亿）。激活比例约3%，意味着97%的参数在单次推理中处于休眠状态。

这个稀疏度的工程意义很直接。假设一张国产卡显存64GB，如果1.6万亿参数全部加载，需要数TB显存，根本装不下。但只加载480亿激活参数，配合模型并行和流水线并行，可以在有限显存内完成推理。

但MoE有个天然瓶颈：当稀疏度超过95%之后，继续堆专家数量的收益会急剧下降。LongCat-2.0的97%稀疏度已经逼近这个极限。美团团队的选择是——**不在MoE上继续内卷，而是另开一条参数扩展路径**。

这就是N-gram Embedding的由来。

## 第二层：LSA——给注意力机制做减法

LSA（LongCat Sparse Attention，LongCat稀疏注意力）是LongCat-2.0最硬核的技术创新。它的目标只有一个：让模型在100万token的上下文窗口下，注意力计算不从平方级爆炸。

传统Transformer的注意力机制有个致命问题：计算复杂度是O(n²)。1000个token的注意力计算量是100个token的100倍。当上下文膨胀到100万token时，这个平方增长会让算力和显存同时崩溃。

LSA通过三层正交设计来解决这个问题。

**第一层设计：流感知索引（Streaming-aware Indexing，SI）**

核心思想是把内存访问模式从随机变成顺序。注意力计算时，模型需要频繁读取不同位置的Key和Value向量。如果读取位置是随机的，国产卡的内存带宽会成为瓶颈。SI通过把硬件对齐的连续数据读取和动态随机选择结合，重构了token选择流水线，把碎片化的内存访问转化为高度可预测的流式读取。

简单说：以前是东一块西一块地找数据，现在是一口气顺着一个方向读，中间跳过不重要的部分。

**第二层设计：跨层索引（Cross-layer Indexing，CI）**

不同Transformer层对注意力的需求不一样。浅层更关注局部语法结构，深层更关注全局语义关联。CI让相邻层共享注意力索引，避免每层重复计算哪些token该被关注。这个设计在长文本场景下尤其有效——当上下文里有大量重复或冗余信息时，跨层复用索引能显著降低计算量。

**第三层设计：内存压缩**

在前两层的基础上，LSA对KV Cache（记忆缓存）做动态压缩。对于长文本中信息量低的段落，压缩其KV表示的精度；对于关键段落，保留完整精度。这种自适应精度策略在不损失模型效果的前提下，进一步降低显存占用。

```
LSA三层优化效果对比
├─ 传统全注意力：O(n²) 计算量，1M上下文不可行
├─ LSA + SI：     内存访问从随机→顺序，带宽利用率提升
├─ LSA + SI + CI：跨层复用索引，重复计算减少
└─ LSA完整版：    计算量接近O(n)，1M上下文可落地
```

从美团技术博客披露的数据看，LSA使得LongCat-2.0在100万token上下文下的推理效率，相比传统全注意力提升了一个数量级。

## 第三层：N-gram Embedding——稀疏度极限下的新参数路径

前文提到，LongCat-2.0的MoE稀疏度已经接近97%，继续堆专家的收益有限。美团团队的解法很有意思：**不增加专家，而是增加一个独立的N-gram Embedding模块**。

N-gram是NLP里的经典概念，指连续的N个token组成的片段。比如"深度学习模型"这个句子里，"深度"是一个2-gram，"深度学习"是一个3-gram。N-gram Embedding的核心思想是：模型不仅学习单个token的表示，还学习token组合的表示。

LongCat-2.0在这个模块上投入了135B参数，占总参数不到10%。这些参数不参与MoE路由，而是作为一个独立的语义增强层，专门强化代码片段级的理解能力。

为什么是代码？因为代码有很强的局部结构特征。一个函数定义、一个循环体、一个条件判断，本质上都是N-gram模式。N-gram Embedding能让模型更好地捕捉这些局部代码结构，而不需要依赖长距离注意力。

这个设计的巧妙之处在于**与MoE的正交性**。MoE负责全局知识路由，N-gram Embedding负责局部语义增强，两者互不干扰。即使MoE的稀疏度已经很高，N-gram模块仍然能独立发挥作用。

## 训练数据与后训练：为Agentic Coding而生

LongCat-2.0的预训练数据规模超过30万亿token（部分来源称35万亿），覆盖中英多语言语料、开源代码库、工具调用样本和智能体交互数据。

代码数据是重点。训练阶段引入了超过10万亿token的真实开源项目代码，覆盖Python、Java、C++等12种主流编程语言。同时构建了超过100万个调试会话的交互数据集，让模型学会分析错误和修正代码。

后训练阶段采用**多教师在线蒸馏**。模型将专家分为三类：Agent（智能体决策）、推理（逻辑推导）、交互（用户对话）。每类专家有专门的教师模型指导，最终融合成一个统一的输出。

这种分类蒸馏的针对性很强。Agent类专家负责理解任务目标并分解执行步骤，推理类专家负责中间的逻辑推导，交互类专家负责把结果翻译成用户能理解的表达。三类分工协作，而不是一个通用模型包揽全部。

## 国产算力适配：五万卡的工程账

LongCat-2.0的训练算力配置值得关注。

根据公开信息，训练核心算力约五万张昇腾910/950，此外还有寒武纪MLU 580/590参与千卡级灰度测试，天数智芯参与部分推理业务，摩尔线程和沐曦适配中小模型与推理场景。

多厂商并行适配的策略说明了一件事：美团在做国产算力生态的**去单一化**。不绑定任何一家国产芯片厂商，而是同时适配多家，降低供应链风险。

推理端的挑战更大。昇腾A2单卡显存仅64GB，要部署1.6万亿参数模型并支持百万级长序列，需要精细的并行策略。华为CANN团队采用PD分离架构（Prefill-Decode分离），把预填充阶段和解码阶段分配到不同的计算节点，分别优化。

```
国产算力适配矩阵
├─ 训练：华为昇腾910/950 × 5万张（主力）
├─ 灰度测试：寒武纪MLU 580/590（千卡级）
├─ 推理：天数智芯（部分业务）
├─ 中小模型：摩尔线程、沐曦（投资+适配）
└─ 部署策略：PD分离 + 模型并行 + 流水线并行
```

## 数据验证与行业定位

| 指标 | LongCat-2.0 | 对比基准 |
|:---|:---|:---|
| 总参数 | 1.6T | GLM-5.2: 744B |
| 激活参数 | ~48B (动态33B~56B) | GLM-5.2: ~40B |
| 上下文长度 | 原生1M tokens | GLM-5.2: 1M tokens |
| 预训练数据 | 30-35T tokens | DeepSeek-V4: 数十T |
| 开源协议 | MIT | GLM-5.2: MIT |
| 训练算力 | 5万张国产卡 | 多数模型依赖英伟达 |

LongCat-2.0的参数规模几乎是GLM-5.2的两倍，但单次激活参数只多了约20%。这说明美团在MoE路由效率上做了更激进的优化，用更大的总参数量换取更高的知识容量，同时控制激活成本。

但参数大不等于能力强。LongCat-2.0的真正价值在于证明了一件事：**国产算力集群已经具备训练万亿参数模型的工程可行性**。从2023年LongCat团队开始探索国产算力，到2026年开源万亿模型，这条路线走了三年。

## 一句话核心判断

LongCat-2.0不是国产AI模型里最强的，但它是第一个在国产算力上跑通万亿参数全流程的。这个第一名的意义不在于参数数字，而在于它给整个行业打了一个样——证明国产芯片+国产框架+国产模型，这个闭环是可以走通的。
